# タスクワークスペース

## タスクワークスペースとは

各タスクで実装を行う環境

自動チェックを含めてタスクで行われるすべてのことはタスクワークスペースで行われる

## フロー

### 初期化

タスクが実施前から計画中に遷移するタイミングで、プロジェクトワークスペースと同等の情報を持ったタスクワークスペースを作成する

このタイミングでタスクワークスペース専用のgit branchが切られる
git branchはタスク毎にユニークである必要がある
ユーザーが作業ブランチを指定した場合はそのgit branchを利用し、未指定の場合はタスクごとにユニークなgit branchを自動生成する

- 作業ブランチ未指定の場合は system managed branch として扱う
  - レビュー後にベースブランチへマージする対象になる
- 作業ブランチ指定の場合は user managed branch として扱う
  - レビュー後もブランチを保持し、ベースブランチへの自動マージ対象にはしない
  - このモードは作業ブランチからベースブランチへの PR 作成を意図する

### 完了

タスクが完了したら、review merge policy が `merge_to_base` の場合はプロジェクトワークスペースで使われているgit branchにタスクワークスペースのgit branchをマージする
review merge policy が `preserve_branch` の場合は、タスクワークスペースのgit branchを残したままタスクを完了する

もしコンフリクトが発生していたらユーザーにコンフリクトの解消を依頼する
