# nami

`nami` は、Cline を複数タスクのチャット UI として扱うためのデスクトップアプリです。

リポジトリをワークスペースとして選び、やりたい作業をタスクとして登録すると、Cline の plan / act の流れ、実行ログ、承認待ち、差分確認、自動チェックまでを一つの画面で追えます。複数の作業を同時に進めたいときや、AI エージェントの作業を人間がレビューしながら安全に進めたいときに向いています。

## nami でできること

- **タスクボードで作業を整理**
  - 未開始、計画中、実行中、レビュー待ち、完了などの状態ごとにタスクを一覧できます。
- **チャットで Cline に指示**
  - タスクごとに会話履歴を保持し、追加指示や修正依頼をその場で送れます。
- **plan / act の流れを画面上で制御**
  - まず計画を作らせ、内容を見てから実行へ進めるか、計画を練り直すかを選べます。
- **承認リクエストとツール実行ログを確認**
  - AI の応答だけでなく、ツール呼び出し、承認要求、エラー、自動チェック結果もタイムライン上で確認できます。
- **タスクごとの作業ブランチを管理**
  - タスク作成時に作業ブランチを指定できます。未指定の場合はタスク用のブランチが自動で扱われます。
- **タスク間の依存関係を設定**
  - あるタスクが終わってから次のタスクを進める、という順番を UI から管理できます。
- **自動チェックを実行**
  - ワークスペースごとにテストや lint などのチェックコマンドを設定できます。タスク完了後にチェックし、失敗した場合は結果を Cline にフィードバックできます。
- **レビューと commit をまとめて実行**
  - レビュー待ちのタスクでは、変更差分を確認し、commit メッセージを入力して commit できます。

## 基本的な使い方

1. `Workspace` から作業したいリポジトリを選びます。
2. `新しいタスクを作る` から、Cline に任せたい作業内容を入力します。
3. 必要に応じて、作業ブランチや依存タスクを設定します。
4. タスクを開き、計画を開始します。
5. 計画内容を確認し、問題なければ実行へ進めます。修正したい場合は計画を練り直します。
6. 実行中はチャット、ツール実行、承認待ち、エラーなどをタイムラインで確認します。
7. レビュー待ちになったら、差分を確認して commit します。

## 便利な設定

設定は、ワークスペースを選択した状態で `設定` から開けます。

### 自動承認

計画が完了したあと、人間の「実行に移す」操作を待たずに act モードへ進める設定です。

計画を毎回必ず確認したい場合は OFF のまま使ってください。定型的な作業や、細かく分割した安全なタスクでは ON にすると流れが速くなります。

### 自動チェック

タスク完了後に実行するチェックコマンドを設定できます。

例:

```bash
npm run test
```

複数ステップを登録できるため、テスト、型チェック、lint などを順番に実行できます。チェックに失敗した場合は、失敗したコマンドと出力を確認できます。

## インストールと起動

このリポジトリは現在、ソースから起動またはビルドして使う構成です。

### 必要なもの

- Node.js
- npm
- macOS

macOS 向けのパッケージ設定が用意されています。

### 依存関係のインストール

```bash
npm install
```

### 開発モードで起動

```bash
npm run dev
```

Vite と Electron を起動し、ローカルでアプリを試せます。

### アプリをビルド

```bash
npm run build
```

このコマンドは次の処理を順番に実行します。

- React 側のビルド
- Electron 側の TypeScript ビルド
- macOS 向けアプリのパッケージング

生成物は `release/` 配下に出力されます。現時点の設定では macOS の directory target を使うため、配布用のインストーラではなく `.app` 形式のアプリディレクトリが作られます。

## よく使うコマンド

```bash
# 開発モードで起動
npm run dev

# アプリをビルド
npm run build

# テストを実行
npm run test

# 型チェック
npm run typecheck

# lint
npm run lint
```

## こんな使い方に向いています

- 複数の AI コーディングタスクを同時に管理したい
- Cline の作業を plan / act に分けてレビューしながら進めたい
- 変更内容を commit 前に UI で確認したい
- タスク完了後にテストや lint を自動で走らせたい
- 依存関係のある作業を順番に進めたい

## 開発者向け情報

内部設計や実装ルールは、次のドキュメントにまとまっています。

- `AGENTS.md`
- `eslint.architecture.config.js`
- `docs/archutecture.md`
- `docs/modeling/`

ディレクトリの大まかな役割は次の通りです。

- `electron/`: Cline 操作、IPC、永続化、ドメインロジック
- `src/`: React ベースの UI
- `share/`: Electron と React の間で共有する型
- `docs/`: アーキテクチャやモデリングの設計資料
