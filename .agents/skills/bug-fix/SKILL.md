# bug fix

バグ修正のためのノウハウをまとめたもの

## 仮説検証のために小さいサンプルコードを作ること

- 最初から検証のためにutを書いてしまうと、原因ではなかった場合に負債になってしまう
- 確実に原因といえるものがわかるまでは小さいサンプルコードを実行し、確証を得るまでutは書かないこと
- 実際に動かしてないコードは仮説とずれる可能性があるので、サンプルコードでの仮説検証は最下流まで徹底的にやること

## 確証を得たあとはユーザーに承認を取る

- 原因が明らかになったら原因の詳細、修正に必要な情報、追加すべきテストをまとめ、ユーザーに承認を得ること
- ユーザーの承認なしにプロダクトコードに修正を入れることを禁じる
